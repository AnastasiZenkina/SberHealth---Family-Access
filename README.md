## Решение
Я спроектировала и реализовала микросервис, который:
- Добавляет роли: `PARENT` (родитель), `CHILD` (ребёнок), `GUARDIAN` (опекун).
- Проверяет, может ли пользователь заказать услугу для другого члена семьи.
- Даёт фронтенду список «за кого можно заказывать», чтобы интерфейс показывал подсказку «Вы оформляете заказ для Ивана».
- Использует Spring Boot и базу данных H2 (в памяти) для демонстрации


## Архитектура
Хранение данных:
Profile - профиль пользователя
FamilyMember - другие члены семьи в профиле
FamilyGroup - семейная группа
FamilyRole - роли членов семьи

Репозитории:
ProfileRepository
FamilyRoleRepository
FamilyMemberRepository
FamilyGroupRepository - базовые круд операции для всех таблиц в базе 

Бизнес логика:
FamilyService - методы для создания группы, добавления человека, получения списка всех людей, проверки прав и возврата списка профилей за кого пользователь может делать заказ
для фронтенда, чтобы в последствии выводить всплывающую надпись "Вы делаете заказ для ..."

Метод проверки canOrderFor проверяет, есть ли у заказчика роль «родитель» или «опекун» в той же группе, где находится пользователь с ролью «ребёнок» (или «опекун») 
А метод getAllowedTargets собирает список всех людей (детей/опекаемых), за которых данный пользователь может оформлять заказы, и возвращает их без повторов

Контроллер:
FamilyController - содержит методы и эндпоинты API для создания группы, добавления человека, получения списка людей, проверки прав и получения списка тех, за кого можно заказывать

Дополнительный системный класс для запуска DataLoader проверяет, есть ли роли в базе. Если пусто — создаёт три роли: Родитель (может заказывать за других), Ребёнок (за него можно заказывать), Опекун (может и то, и другое)


## Технологии
- Java 17
- Spring Boot 3.5.14
- Spring Data JPA
- H2 Database
- Maven
- Lombok (для чистоты кода)

## Как запустить
1. Склонируйте себе репозиторий
2. Перейдите в папку проекта и запустите на порту 8080:  
   `cd family-access`

## Примеры API (можно протестировать в терминале)

В H2-консоль:
Зайдите на http://localhost:8080/h2-consol
- JDBC URL: `jdbc:h2:mem:testdb`
- User: `sa`, пароль пустой

Выполните SQL:
`sql
INSERT INTO profile (full_name, date_of_birth) VALUES ('Иван Иванов', '1985-03-15');
INSERT INTO profile (full_name, date_of_birth) VALUES ('Петя Иванов', '2015-08-20');
INSERT INTO profile (full_name, date_of_birth) VALUES ('Маша Иванова', '2018-11-02');
`

В консоли Idea
Создать семейную группу
`curl -X POST "http://localhost:8080/api/family/groups?name=%D0%98%D0%B2%D0%B0%D0%BD%D0%BE%D0%B2%D1%8B&creatorId=1"`

Добавить ребенка
`curl -X POST "http://localhost:8080/api/family/groups/1/members?profileId=2&roleName=CHILD&adderId=1"`

Проверить, может ли родитель заказать для ребёнка
`curl "http://localhost:8080/api/family/check?customerId=1&targetId=2"`

Проверить, может ли ребёнок заказать для родителя
`curl "http://localhost:8080/api/family/check?customerId=2&targetId=1"`

Получить список тех, для кого можно заказывать
`curl "http://localhost:8080/api/family/allowed-targets?customerId=1"`

Показать всех членов группы
`curl "http://localhost:8080/api/family/groups/1/members"`
